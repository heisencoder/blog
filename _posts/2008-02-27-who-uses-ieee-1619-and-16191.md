Last December, <a href="http://www.ieee.org/portal/site">IEEE</a> approved the standards two encryption standards (See <a href="http://standards.ieee.org/announcements/StdsForEncryption.html">Press Release</a>):<br />
<ul>
<li>IEEE 1619 (specifying the XTS encryption mode, commonly used for disk encryption); and</li>
<li>IEEE 1619.1 (specifying GCM, CCM, CBC-HMAC, and XTS-HMAC encryption modes, typically for tape) .</li>
</ul>
Byte and Switch <a href="http://www.byteandswitch.com/document.asp?doc_id=141444">interviewed me in an article</a> discussing this and other related standards.<br />
<br />
Currently the <a href="http://siswg.net/">IEEE Security in Storage Working Group</a> (SISWG) is investigating the possibility of submitting the IEEE 1619 XTS mode to NIST for consideration as an Approved Mode of Operation for FIPS 140-2 certification.  One of the <a href="http://grouper.ieee.org/groups/1619/email/msg02266.html">questions asked</a> on the <a href="http://grouper.ieee.org/groups/1619/email/">SISWG e-mail reflector</a> was whether there is widespread industry support for these newly approved modes.<br />
<br />
From doing some web searching, I've come up with the following list of companies who are claiming compliance to these newly approved standards.<br />
<br />
<span style="font-weight: bold;">IEEE 1619 (XTS-AES) Support</span><br />
<ul>
<li><a href="http://www.truecrypt.org/">True Crypt</a> - <a href="http://www.truecrypt.org/docs/?s=version-history">Version 5.0 now supports XTS for software disk encryption</a></li>
<ul>
<li>(Update from 2016: TrueCrypt is no longer maintained and has been accumulating security issues so this is no longer recommended.  Please see <a href="https://www.comparitech.com/blog/information-security/truecrypt-is-discoutinued-try-these-free-alternatives/">this list of alternatives instead</a>).</li>
</ul>
<li><a href="http://www.freeotfe.org/">FreeOTFE</a> - Free On-The-Fly-Encryption</li>
<li><a href="http://www.saout.de/misc/dm-crypt/">dmcrypt </a>- Encryption for the Linux 2.6 kernel</li>
<li><a href="http://www.hifn.com/">Hifn</a>: "<a href="http://www.hifn.com/pressReleases.aspx?id=3980&terms=1619&searchtype=1&fragment=False">Hifn’s full line of Applied Services Processors as well as its board-level security acceleration products currently support, and are compliant with, the encryption algorithms specified in the IEEE 1619 standards</a>"</li>
<li><a href="http://www.heliontech.com/">Helion Technology</a>: <a href="http://www.heliontech.com/aes_xex.htm">AES-XTS cores</a></li>
<li><a href="http://www.ellipticsemi.com/">Elliptic Semiconductor:</a></li>
<ul>
<li><a href="http://www.ellipticsemi.com/products-clp-33.php">CLP-33 XTS-AES cipher core</a></li>
<li><a href="http://www.ellipticsemi.com/products-clp-35.php">CLP-35 High Throughput XTS-AES cipher core</a></li>
</ul>
<li><a href="http://www.ipcores.com/">IP Cores</a>: <a href="http://www.ipcores.com/AES_XTS_IP_core.htm">XTS-AES IEEE P1619 Core Families XTS2 and XTS3</a></li>
<li>Hightech Global Design & Distribution: <a href="http://www.hitechglobal.com/IPCores/AES-GCM-XTS-CCM.htm">Combined AES-GCM-XTS/XEX-CCM IP Core</a></li>
<li>JetStream Media Technologies: <a href="http://www.jetsmt.com/us4s/JetXTS_1983460.pdf">High Speed XTS/XEX-AES Core</a></li>
<li><a href="http://www.safenet-inc.com/">SafeNet, Inc.</a>: <a href="http://www.safenet-inc.com/products/ip/safeXcel_IP_AESGCMXTS_Acc.asp">SafeXcel IP AES/GCM/XTS Accelerators</a> <span style="font-family: "arial"; font-size: 85%;"><span style="font-family: "arial"; font-size: 10pt;"><a href="http://www.safenet-inc.com/products/ip/safeXcel_IP_AESGCMXTS_Acc.asp" rel="nofollow" target="_blank"><span class="yshortcuts" id="lw_1209046570_1"></span></a></span></span></li>
</ul>
IEEE 1619.1 (GCM, CCM, CBC-HMAC, XTS-HMAC):<br />
<ul>
<li>IBM, HP, Quantum, Tandberg's LTO-4 Tape Drive uses IEEE 1619.1 GCM-AES</li>
<li>IBM's TS1120 Enterprise Tape Drive uses GCM</li>
<li>Sun's T10000 Enterprise Tape Drive uses 1619.1 CCM</li>
<li>Many core vendors (basically same list as for 1619)</li>
</ul>
I expect that these lists will grow considerably after IEEE officially publishes 1619 and 1619.1 in the next couple months.  Right now, it's hard to assess compliance because a product cannot technically claim compliance until the standard is published.<br />
<br />
If you know of implementations that expect IEEE 1619 or 1619.1 compliance, please post a comment with the vendor and product name, with a link to the appropriate webpage.
